Approach

Two hats to wear at once: customer (what would real users ask) and SME (what tests the failure modes deterministically). The customer hat keeps the cases realistic — questions someone might actually type into a Wikipedia-grounded QA tool. The SME hat keeps each case diagnostic — it has to fail in a specific, attributable way if the system is broken in a specific way.

A few constraints I'd hold:

  • Each case targets one failure mode. Cross-category contamination dilutes signal. A question that's both multi-hop AND has a buried answer makes a failure ambiguous.
  • Avoid Einstein/Nobel in actual eval cases — it's already in the system prompt as the worked example, which would leak into the model's behavior unfairly.
  • Wikipedia-stable answers. Avoid cases where the answer might change between when I write it and when we run eval (population figures, current officeholders go in temporal, not simple_factual).
  • Difficulty mix per category. Inside simple_factual, vary subject matter (history, geography, biography, science) so passes are robust, not just "the model knows the same trivia category."

I'd write the dataset incrementally — propose one or two candidates per category for your review, iterate on phrasing and gold answers together, then I extend each category to its target count once we've calibrated the bar.

Below: candidate questions per category, with a short note on what each one is meant to test. Customer voice intentional — phrased the way a real user might type, not how a test harness would prompt.

simple_factual (target: 4, easy) — single canonical fact, well-documented in Wikipedia lead, no real ambiguity.

  • "When was the Eiffel Tower completed?" → 1889. Clean date question, lead-of-article fact.
  • "Who wrote Pride and Prejudice?" → Jane Austen. Tests the most basic case.

What I want here: cases the system MUST get right. Failures here mean something fundamental is broken.

multi_hop (target: 5-6, medium-hard) — sequential A→B→C, must chain.

  • "What is the official language of the country whose capital hosted the 1972 Summer Olympics?" → Munich → West Germany → German.
  • "Who succeeded the UK Prime Minister who served during the 1940 Battle of Britain?" → Churchill → Attlee.

Critical for these: the answer cannot be findable in a single search of any of the entities. The agent has to actually decompose. I'd test by trying each in search_wikipedia solo — if any one search has the final answer in its lead extract, the case isn't really multi-hop.

multi_source (target: 3, medium) — parallel info from multiple articles.

  • "Who lived longer: Albert Einstein or Stephen Hawking?" → Einstein 76 years vs Hawking 76 years (this might tie, hmm — let me pick another). Maybe "Did Charles Darwin or Isaac Newton live longer?" → Darwin 73, Newton 84.
  • "Which mountain is taller, K2 or Kangchenjunga?" → K2 (8,611m) vs Kangchenjunga (8,586m). Genuinely close numbers force the model to compare carefully.

disambiguation_explicit (target: 3, medium) — search returns a disambig page or implies multiple senses.

"""Verification helper: for each candidate, fetch lead extract AND full article
text. Report whether the candidate answer appears in lead vs body."""

import httpx

UA = "wiki-qa-takehome/0.1 (contact via repo issues)"


def fetch(title: str, intro_only: bool) -> tuple[str, str]:
    """Returns (article_title, extract). intro_only=True for lead only."""
    params = {
        "action": "query",
        "format": "json",
        "prop": "extracts",
        "explaintext": "1",
        "titles": title,
        "redirects": "1",
        "exchars": "20000" if not intro_only else "2000",
    }
    if intro_only:
        params["exintro"] = "1"
    r = httpx.get("https://en.wikipedia.org/w/api.php", params=params,
                  headers={"User-Agent": UA}, timeout=15)
    r.raise_for_status()
    pages = r.json().get("query", {}).get("pages", {})
    if not pages:
        return title, ""
    page = next(iter(pages.values()))
    return page.get("title", title), page.get("extract", "")


def check(label: str, article_title: str, needle_or_pattern: str | list[str]):
    """Fetch lead and body. Report whether the needle appears in each."""
    needles = [needle_or_pattern] if isinstance(needle_or_pattern, str) else needle_or_pattern
    title_lead, lead = fetch(article_title, intro_only=True)
    title_full, full = fetch(article_title, intro_only=False)
    body = full[len(lead):]  # rough body = full minus lead

    print(f"\n=== {label} ===")
    print(f"Article: {title_lead!r}")
    print(f"Lead chars: {len(lead)}, full chars: {len(full)}")
    for needle in needles:
        in_lead = needle.lower() in lead.lower()
        in_body = needle.lower() in body.lower()
        in_anywhere = needle.lower() in full.lower()
        print(f"  needle={needle!r}: in_lead={in_lead}, in_body={in_body}, in_full={in_anywhere}")

import httpx

API = "https://en.wikipedia.org/w/api.php"


def fetch_lead(title: str, exchars: int = 2000) -> tuple[str, str]:
    r = httpx.get(API, params={
        "action": "query", "format": "json", "prop": "extracts",
        "exintro": "1", "explaintext": "1", "exchars": str(exchars),
        "titles": title, "redirects": "1",
    }, headers={"User-Agent": UA}, timeout=15)
    r.raise_for_status()
    pages = r.json().get("query", {}).get("pages", {})
    page = next(iter(pages.values()))
    return page.get("title", title), page.get("extract", "")


def fetch_full_wikitext(title: str) -> str:
    r = httpx.get(API, params={
        "action": "parse", "format": "json", "page": title,
        "prop": "wikitext", "redirects": "1",
    }, headers={"User-Agent": UA}, timeout=20)
    r.raise_for_status()
    data = r.json()
    if "error" in data:
